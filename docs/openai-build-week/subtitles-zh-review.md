# WeakSpot OpenAI Build Week Demo 中文字幕审阅稿

> 与英文旁白逐句对应。中文确认后，必须依据最终英文逐句音频的真实时长生成 SRT；这里的章节时间不是最终字幕时间码。

## 00:00–00:18｜问题与承诺

1. 大多数 AI 语言老师可以修改一次回答，却不知道明天应该再次练习哪个弱点，也不知道哪种练习真正有效。
2. WeakSpot 是一位跨会话英语教练，会把学习证据转化为下一个有用的真实任务。

## 00:18–00:48｜Build Week 新增内容

3. 为了参加 OpenAI Build Week，我在现有产品上新增了 GPT-5.6 自适应任务规划器。
4. 我只需要选择可用时间、打字还是口述，以及当前精力。
5. 调度器会综合技能的薄弱程度、不确定性、复习到期风险、相关性和近期重复次数进行排序。
6. 然后，GPT-5.6 Sol 通过 OpenAI Responses API 接收这份受限的决策上下文，并生成一个结构化任务。

## 00:48–01:18｜可见的 GPT-5.6 证据

7. 这不是静态模板：页面会显示 OpenAI 实际返回的模型、Responses API 和规划证据。
8. 我可以看到为什么现在练这个、使用了哪些证据、任务如何调整，以及教练会观察什么。
9. 输出使用原生 Pydantic Structured Outputs 验证，因此任务和证据说明遵循同一份类型化契约。

## 01:18–01:48｜多样练习与迁移

10. 它可以生成真实角色对话、看图描述与推断、听后复述、情境决策，或真实消息中的词汇应用。
11. 在普通聊天中，WeakSpot 也可以把过去的薄弱项自然放进追问，而不会宣布这是一场测试。
12. 这样检查的是我能否在新语境中独立迁移这项能力，而不是重复刚刚看到的修改答案。

## 01:48–02:13｜有控制权的记忆

13. 每次尝试都会返回证据，包括独立成功、有提示成功、失败、回避、回答方式、任务类型和延迟迁移。
14. 下一次请求不会发送完整历史。
15. WeakSpot 在固定预算内最多召回六条相关记忆，同时我可以查看、修改、置顶或遗忘每一条。

## 02:13–02:40｜Codex 协作与架构

16. Codex 是我的工程开发协作者，而不是嵌入网站的运行时模型。
17. Codex 审查了多提供商系统，核对 OpenAI 指南，设计 Responses 路径，实现功能，并补充契约测试与提交文档。
18. 产品运行时调用 GPT-5.6 Sol；服务器会保护密钥、关闭存储、对安全标识符进行哈希，并阻止误导性的模型标记。

## 02:40–02:56｜证据与结尾

19. 公开健康接口会在不暴露凭证的情况下显示已启用功能、模型和 API；构建日志会把这次扩展与早期工作分开。
20. WeakSpot 不只记住错误；借助 GPT-5.6，它会在正确的时机，把正确的证据变成正确的挑战。
