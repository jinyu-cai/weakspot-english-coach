# Recording Shot List

Record at 1920x1080, 30 fps. Use 125% to 150% browser zoom and a dedicated
test learner. Keep the mouse still unless the narration names the visible
feature.

## Shared demo data

Writing diagnosis input:

> I want to communicate more naturally at work and in everyday life, and I prefer concise feedback. Last year,
> I go to many countries and learn a lot of cultures. It help me understand
> people better.

Recall Preview query:

> Create my next English practice.

Prepare before capture:

- one active everyday/work communication goal;
- one active concise-feedback preference;
- two evidenced weaknesses;
- one strategy card with attempts, average score, and success rate;
- one superseded preference and its active replacement;
- one disposable episode that can be forgotten;
- a completed benchmark result produced by the current revision.

## 01 — Hook, 0:00–0:16

1. Begin on the product home page with no loading state.
2. Hold for two seconds so the product name is readable.
3. Open Memory Center.
4. End on the full Memory Center overview with summary cards visible.

Overlay: `A cross-session English coach powered by Qwen`.

## 02 — Autonomous accumulation, 0:16–0:48

1. Begin with the prepared paragraph already entered in Writing Diagnosis.
2. Click Diagnose.
3. Cut the network wait; resume when the result is complete.
4. Briefly show CEFR, corrected text, and the two identified errors.
5. Open Memory Center.
6. Point to Goal, Preference, and Weakness badges.
7. Point to evidence, confidence, and observation count on one card.

Overlay: `One Qwen response → diagnosis + durable memory candidates`.

## 03 — Bounded recall, 0:48–1:16

1. Begin at Recall Preview with the query already entered.
2. Click Retrieve and cut the network wait.
3. Show the recalled communication goal, feedback preference, and relevant weakness.
4. Point to the token meter and keep `estimated tokens / 700` readable.
5. Point to the Why Recalled component scores.
6. End on the recall trace generated by this request.

Overlay: `Hybrid retrieval · ≤6 memories · 700-token budget`.

## 04 — Replacement and forgetting, 1:16–1:39

1. Switch the memory filter from Active to All/Archived.
2. Show the active preference and its superseded predecessor together.
3. Keep both status labels readable.
4. Return to Active.
5. Click Forget on the disposable episode and confirm the success message.

Do not imply that the Edit button creates a superseded row; use a genuinely
prepared conflict generated with the same canonical key.

Overlay: `Merge · Supersede · Expire · User-controlled forget`.

## 05 — Outcome-driven decision, 1:39–2:07

1. Show a Strategy memory card.
2. Point to attempts, average score, and success rate.
3. Scroll to Next Practice.
4. Point to target skill, exercise format, and policy/reason.
5. Click Start Practice and show the generated task without completing it.

Overlay: `Practice outcomes → strategy evidence → better next action`.

## 06 — Architecture and deployment proof, 2:07–2:34

Capture short, readable shots in this order:

1. Architecture diagram with Browser → Alibaba ECS → Qwen → Memory → DynamoDB.
2. Alibaba Cloud ECS instance summary; hide IPs and account identifiers.
3. Public `/api/v1/health` returning success.
4. Safe application log lines showing `qwen3.7-max` and `qwen3.7-plus`.
5. Model Studio page/configuration showing `text-embedding-v4`, without keys.
6. DynamoDB rows beginning with `MEMORY#` and `MEMTRACE#`, without learner PII.

The application must actually be routed to Alibaba ECS during these captures.

## 07 — Evidence and close, 2:34–2:54

1. Show the freshly completed benchmark terminal output.
2. Highlight Recall@6, stale-memory suppression, budget compliance, and context
   reduction one at a time.
3. Cut to the public GitHub repository landing page.
4. Show the visible MIT license.
5. End on the product home page or logo.

Overlay: `WeakSpot English Coach — Remember what works.`
