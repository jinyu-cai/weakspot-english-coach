# WeakSpot Hackathon Demo 中文字幕审阅稿

> 目标时长：2:54。以下时间码用于审阅叙事结构，不是最终字幕时码。
> 文案确认后，先按句生成新版英文配音，再以每句真实音频时长生成中英文 SRT，避免配音与字幕错位。

## 01｜问题与定位（00:00–00:15）

1. 大多数 AI 英语老师，会在对话结束后忘记你。
2. 每次开始新会话，你都要重新说明目标、偏好和反复出现的困难。
3. WeakSpot 不一样。
4. 它是一位拥有持久记忆的英语教练，会跨会话了解你的弱点，也会记住哪种练习方式对你真正有效。

## 02｜从一次写作开始积累记忆（00:15–00:35）

5. 我可以从粘贴一小段英文开始。
6. Qwen 在诊断语言问题的同时，也会识别未来仍然有用的信息。
7. WeakSpot 自动保存我希望在工作和日常交流中更自然表达的目标、简洁反馈偏好，以及有证据支持的时态和主谓一致弱点。
8. 不需要单独填写学习档案，也不需要额外执行一次记忆流程。

## 03｜不止写作：多种真实练习（00:35–01:30）

9. 但粘贴写作只是其中一个入口。
10. 在普通文字或语音对话中，教练会先回应我当前真正想表达的内容，让交流自然继续。
11. 当对话恰好出现合适的语言产出机会时，它才会把一个需要复习的弱点自然放进追问里，却不会告诉我正在接受测试。
12. 这样验证的是我能否在新语境中独立运用，而不是能否背出刚看过的修改答案。
13. 如果我不知道今天该练什么，只要选择可用时间、回答方式和当前精力。
14. 教练会结合学习记忆，自动安排最有用的任务；我也可以主动指定任务形式。
15. 任务包括真实场景对话、看图表达、听后复述、情境决策和词汇应用。
16. 在看图表达中，我需要观察一幅产品内置插图，用英文描述细节、作出推断，或者讲出一段故事。
17. 在听后复述中，我只能播放一段原创英文录音有限次数，再用自己的话复述重点或作出回应。
18. 在情境决策和词汇应用中，系统会给出身份、对象、目标和限制，让我写下或说出在现实中可能使用的话。
19. 聊天首页还能随时生成新的角色场景，并尽量避开最近练过的场景类型，让普通聊天本身也成为持续变化的练习。

## 04｜记忆检索与用户控制（01:30–01:56）

20. 这些练习结果会回到同一套学习记忆中，记录弱点是否能在不同场景和不同回答方式下真正迁移。
21. 新会话不会把全部历史重新塞给模型。
22. Qwen text-embedding-v4 与关键词匹配只召回当前最相关的内容；最终 Memory Pack 不超过六条，并严格限制在七百个 token 以内。
23. 相同观察会合并，冲突信息会被替换并归档，不同记忆也会按规则衰减或过期。
24. 学习者始终可以查看、更正、置顶或忘记任何一条记忆。

## 05｜记住什么练习真正有效（01:56–02:18）

25. 记忆不仅保存事实，也会改善教练下一步的决策。
26. 每次练习都会更新特定弱点、任务形式和提示程度的效果证据。
27. 下一项任务不是简单选择最低分，而是综合掌握差距、近期错误、间隔复习、历史效果、合适难度和探索价值。
28. 推荐结果还会解释为什么现在练这个，并链接回支持这项判断的记忆。

## 06｜Qwen 与 Alibaba Cloud 架构（02:18–02:40）

29. 应用由 Next.js 前端和 FastAPI 后端组成，通过 Docker 运行在 Alibaba Cloud ECS 上。
30. Qwen 3.7 Max 负责深度分析与规划，Qwen 3.7 Plus 负责快速交互，Qwen text-embedding-v4 为混合记忆检索生成向量。
31. DynamoDB 保存跨会话记忆、生命周期信息和可审计的召回记录。

## 07｜证据与结尾（02:40–02:54）

32. 在可复现基准测试中，WeakSpot 的 Recall at Six 达到百分之百，同时过滤失效和被替换的记忆，遵守全部 token 预算，并将样例上下文缩短百分之八十二点六。
33. WeakSpot English Coach 不只记住你的错误，也记住你的目标，以及什么方法最适合你。

## 事实边界

- “语音”当前指语音输入转成文字后确认，以及 AI 语音播放；视频不宣称已实现声学层面的发音评分。
- “看图表达”使用产品内置、允许列表中的插图；视频不宣称模型对任意上传图片进行机器视觉核验。
- “听后复述”使用原创脚本或用户有权使用的字幕文本；视频不宣称系统会抓取或下载外部媒体。
