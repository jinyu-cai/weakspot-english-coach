# UI Spotlight Preview（本地/分支预览）

> **状态**：仅用于验证与设计评审。
> **分支**：`ui/learning-spotlight-preview`
> **边界**：不合并 `main`、不部署生产、不改后端与数据生命周期。

## 设计主题：聚光灯学习台（Spotlight Study Desk）

目标：用户打开应用后，**立刻知道现在该做什么**；界面有辨识度，但不花哨、不抢主任务。

### 核心原则

1. **先行动，后解释**：首页先给出 3 条清晰路径，再展开具体表单。
2. **一条主推荐**：默认推荐「让教练安排今日任务」，其他路径并列可选。
3. **学习闭环可见**：侧边栏固定 `发现 → 练习 → 记住`，帮助用户理解产品节奏。
4. **功能不删减**：导航入口、诊断、对话、计划等全部保留。
5. **验证通过后再合并/部署**。

## 本预览改了什么

| 区域 | 变化 |
| --- | --- |
| 视觉系统 | 暖色纸张背景 + 墨水色主按钮 + 琥珀色点缀；更圆的卡片与 spotlight 阴影 |
| 侧边栏 | 增加学习闭环条、主入口「开始」标签、更清晰的分组与图标底板 |
| 顶栏 | 当前页说明保留；今日任务页显示「开始」提示 |
| 首页 `/` | 聚光灯入口：路径 A 教练 / B 写作诊断 / C 对话；下方保留完整诊断输入 |
| 诊断输入 | 更明确的「可分析」状态、示例按钮与主按钮层级 |
| 文案 | 中英双语同步：`现在做什么`、路径说明、闭环文案 |

## 未改动的边界

- 后端 API、DynamoDB、MemoryAgent 算法
- 部署域名 / Cloudflare / Vercel 生产流量
- 不 merge 到 `main`（需你验证通过后再决定）

## 本地预览

```bash
cd apps/web
pnpm install
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000 pnpm dev
```

打开首页，重点检查：

1. 是否一眼看出「现在可以怎么开始」
2. 路径 A/B/C 是否语义清楚
3. 侧边栏闭环是否有帮助、是否太吵
4. 手机宽度下卡片与按钮是否完整可读
5. 中英文切换是否一致
